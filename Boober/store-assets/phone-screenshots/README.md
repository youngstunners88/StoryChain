# Phone Screenshots Placeholder

The following screenshots are needed for Google Play Store submission:

## Required Screenshots (3-8)

Google Play requires 3-8 phone screenshots. Recommended:

1. **onboarding-screen.png** - Welcome/onboarding screen showing role selection (Passenger/Driver/Marshal)
2. **home-dashboard.png** - Main dashboard with nearby taxi ranks
3. **rank-status.png** - Real-time taxi rank status with queue information
4. **driver-profile.png** - Driver verification profile with rating
5. **safety-features.png** - Trip sharing and emergency contacts
6. **map-view.png** - Interactive map showing taxi ranks

## Screenshot Specifications

- **Phone screenshots**: 16:9 or 9:16 aspect ratio
- **Minimum**: 320px width
- **Maximum file size**: 8MB per screenshot
- **Formats**: PNG or JPEG

## How to Capture

Since Boober is currently a web app (React + Vite), you need to:

1. **If using Capacitor**: Run on device/emulator and capture screenshots
2. **From browser**: 
   - Open https://boober.app or local dev server
   - Use browser DevTools → Device Mode
   - Capture screenshots at phone dimensions (e.g., 375×812 for iPhone, 360×640 for Android)
3. **Design team**: Create high-quality mockups in Figma/Canva
