## 🔴 Critical Bugs (MUST FIX BEFORE LAUNCH)

### 1. Backend Content Moderation Not Enforced
**Severity:** CRITICAL  
**Location:** `backend/server.py` - `/posts` endpoint  
**Issue:** The backend accepts posts containing links and explicit content, even though the frontend has content moderation. This allows users to bypass moderation by calling the API directly.

**Proof:**
```bash
curl -X POST http://localhost:8000/api/posts \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"content":"https://google.com"}'
# Returns: 201 Created (should be blocked)
```

**Fix:** Add content validation in the backend before saving posts/messages.

### 2. Registration Fails Without Phone Number
**Severity:** CRITICAL  
**Location:** `backend/server.py` - `/auth/register` endpoint (lines 370-375)  
**Issue:** Registration with email only fails because the duplicate check query uses `{"$or": [{"email": user.email}, {"phone": user.phone}]}`, which matches existing users with `null` phone numbers.

**Proof:**
```bash
curl -X POST http://localhost:8000/api/auth/register \
  -d '{"name":"Test","email":"new@test.com","password":"pass123","role":"PASSENGER"}'
# Returns: {"detail":"User already exists"} - FALSE POSITIVE
```

**Fix:** Only check provided fields:
```python
query = {}
if user.email:
    query["email"] = user.email
if user.phone:
    query["phone"] = user.phone
if query:
    existing = await db.users.find_one(query)
```
