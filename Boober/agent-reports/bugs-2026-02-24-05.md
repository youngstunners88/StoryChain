## 🔴 Critical Bugs (Must Fix Before Launch)

### 1. Registration with Email Always Fails
**Severity:** 🔴 Critical  
**Location:** Backend `server.py` line ~375  
**Issue:** Registration with email always returns "User already exists" even for new emails that don't exist in the database.  
**Reproduction:**
```bash
curl -X POST http://localhost:8000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"name":"NewUser","email":"newuser123@unique.com","password":"test123","role":"PASSENGER"}'
# Returns: {"detail":"User already exists"}
```
**Root Cause:** The query `{"$or": [{"email": user.email}, {"phone": user.phone}]}` likely fails because `user.email` is `Optional[EmailStr]` and may not be properly validated or passed.  
**Workaround:** Use phone number for registration (works correctly).

### 2. Hardcoded JWT Secret in Production
**Severity:** 🔴 Critical  
**Location:** Backend `server.py` line 21  
**Issue:** Default JWT_SECRET hardcoded: `'boober-super-secret-key-change-in-production'`  
**Risk:** If JWT_SECRET env var is not set, all tokens can be forged.  
**Fix Required:** Ensure `JWT_SECRET` environment variable is set in production.

### 3. Hardcoded API URLs in Frontend

**Location:** `src/lib/api.ts` line 1, `src/hooks/useWebSocket.ts`  
**Issue:** API_BASE_URL defaults to `'http://localhost:8000/api'`
