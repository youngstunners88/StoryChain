## 🔴 Critical Bugs (Must Fix Before Launch)

### 1. JWT_SECRET uses placeholder in production
**File:** `.env.production`  
**Issue:** JWT_SECRET is set to `CHANGE_ME_TO_SECURE_RANDOM_STRING` - a literal placeholder string that poses severe security risk.  
**Severity:** CRITICAL  
**Fix:** Generate secure random string: `openssl rand -base64 32`

```env
# Current (INSECURE)
JWT_SECRET=CHANGE_ME_TO_SECURE_RANDOM_STRING
```

### 2. Backend crashes without clear error when MongoDB unavailable
**File:** `backend/server.py`  
**Issue:** When MongoDB is not running, the backend fails with generic "Internal Server Error" - no graceful degradation or clear error message.  
**Reproduction:** Stop MongoDB, try to register → `500 Internal Server Error`  
**Severity:** CRITICAL  
**Fix:** Add try-catch around DB connection and return meaningful error.

## 🟡 Medium Bugs (Should Fix)

### 3. No password strength validation on registration
**File:** `backend/server.py` - `UserCreate` model  
**Issue:** Accepts any password including single character "12". No minimum length enforcement.  
**Severity:** MEDIUM  
**Fix:** Add Pydantic validator for password min length (e.g., 6+ characters)

### 4. Deprecated FastAPI event handlers
**File:** `backend/server.py:1073`  
**Issue:** Uses `@app.on_event("shutdown")` which is deprecated in favor of lifespan handlers.  
**Severity:** MEDIUM  
**Fix:** Migrate to lifespan context managers per FastAPI docs.

### 5. Rate limit too restrictive for testing
**File:** `backend/server.py:386`  
**Issue:** 3 registrations per minute is too low for normal testing/usage.  
**Severity:** MEDIUM  
**Fix:** Increase to 10/minute or implement progressive limiting.
