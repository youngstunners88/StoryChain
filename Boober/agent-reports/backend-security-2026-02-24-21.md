## Detailed Findings

### 1. MongoDB Indexes ⚠️

**Current State**: Only `_id` indexes exist on `users` and `pings` collections.

**Missing Indexes**:
- `users.email` - for login lookups
- `users.phone` - for phone-based auth
- `users.role` - for role-based queries
- `routes.origin_id`, `routes.destination_id` - for route lookups
- `rank_statuses.rank_id` - for rank queries
- `pings.passenger_id`, `pings.status` - for active pings
- `reviews.target_id` - for user reviews

**Impact**: Queries will do full collection scans, causing slow performance at scale.

**Recommendation**: Create indexes immediately:
```javascript
db.users.createIndex({ "email": 1 }, { unique: true })
db.users.createIndex({ "phone": 1 })
db.users.createIndex({ "role": 1 })
db.routes.createIndex({ "origin_id": 1, "destination_id": 1 })
db.rank_statuses.createIndex({ "rank_id": 1 }, { unique: true })
db.pings.createIndex({ "passenger_id": 1 })
db.pings.createIndex({ "status": 1 })
db.reviews.createIndex({ "target_id": 1 })
```

### 2. Rate Limiting ⚠️

**Current State**: `slowapi` Limiter is initialized but **no `@limiter.limit()` decorators** are applied to any endpoints.

**Code Found**:
```python
limiter = Limiter(key_func=get_remote_address)
# ... but no @limiter.limit() on endpoints
```

**Impact**: API is vulnerable to abuse, DoS attacks, and brute force attempts.

**Recommendation**: Add rate limits to sensitive endpoints:
```python
@api_router.post("/auth/login")
@limiter.limit("10/minute")
async def login(request, ...):
    ...

@api_router.post("/pings")
@limiter.limit("30/minute")
async def create_ping(...):
    ...
```

### 3. Input Validation 🟡

**Current State**:
- Email: ✅ Uses `EmailStr` from Pydantic
- Password: ❌ No minimum length validation
- Phone: ❌ No format validation

**Recommendation**: Add password validation in `UserCreate`:
```python
class UserCreate(UserBase):
    password: str = Field(..., min_length=8, max_length=100)
    phone: Optional[str] = Field(None, pattern=r"^\+27[0-9]{9}$")
```
